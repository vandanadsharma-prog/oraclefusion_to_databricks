Oracle Datatype Reference · complete by version

All core datatypes from Oracle5 to 23ai · single table with categories
Category Datatype Description Version introduced / key changes
Character CHAR(size) Fixed‑length character data. Max size is 2000 bytes (or 2000 characters in later versions). Oracle5/6 variable‑length, max 254 chars
Oracle7 fixed‑length, max 255 chars
Oracle8/8i+ max increased to 2000
Character NCHAR(size) Fixed‑length Unicode character data. Max size determined by national character set (max 2000 bytes). Oracle8/8i onwards
Character VARCHAR2(size) The standard variable‑length string datatype. Max size depends on MAX_STRING_SIZE setting. Oracle5/6 max 2000 chars
Oracle7 max 2000 chars
Oracle8/8i‑11g max 4000 bytes/characters
Oracle12c+ can be 32767 bytes/char if MAX_STRING_SIZE=EXTENDED
Character NVARCHAR2(size) Variable‑length Unicode character data. Max size determined by national character set. Oracle8/8i onwards
Character VARCHAR Currently synonymous with VARCHAR2. Avoid using it; future behavior may change. Oracle7 onwards (but VARCHAR2 recommended)
Numeric NUMBER(p,s) Primary numeric datatype. Stores integers, fixed‑point, floating‑point. High precision (38 decimal digits). Present in all versions In Oracle5/6 it was NUMBER(m,n)
Numeric FLOAT(p) Subtype of NUMBER. Floating‑point with binary precision p (1‑126). At least Oracle7 onwards
Numeric BINARY_FLOAT 32‑bit, single‑precision floating‑point (IEEE standard). Oracle 10g onwards
Numeric BINARY_DOUBLE 64‑bit, double‑precision floating‑point (IEEE standard). Oracle 10g onwards
Date / time DATE Stores date and time (century, year, month, day, hour, minute, second). Does not include fractional seconds or time zone. Present in all versions
Date / time TIMESTAMP(p) Extends DATE to include fractional seconds with precision p (0‑9). Oracle9i onwards
Date / time TIMESTAMP WITH TIME ZONE Like TIMESTAMP, plus a time zone displacement. Oracle9i onwards
Date / time TIMESTAMP WITH LOCAL TIME ZONE Normalized to database time zone on storage; adjusted to session time zone on retrieval. Oracle9i onwards
Date / time INTERVAL YEAR TO MONTH Stores a period in years and months. Oracle9i onwards
Date / time INTERVAL DAY TO SECOND Stores a period in days, hours, minutes, seconds. Oracle9i onwards
LOB / binary BLOB Binary Large Object. Max size: (4 GB - 1) × DB block size. Oracle8/8i onwards
LOB / binary CLOB Character Large Object (uses database character set). Max size: (4 GB - 1) × DB block size. Oracle8/8i onwards
LOB / binary NCLOB National Character Large Object (uses national character set). Max size: (4 GB - 1) × DB block size. Oracle8/8i onwards
LOB / binary BFILE Locator to an external binary file. Max external file size: 4 GB. Oracle8/8i onwards
LOB / binary RAW(size) Variable‑length binary data. Max size 2000 bytes; in 12c+ can be 32767 if MAX_STRING_SIZE=EXTENDED. Oracle5/6 max 2GB? (as RAW & LONG RAW)
Oracle7 max 2000 bytes
LOB / binary LONG Variable‑length character data, max 2GB. For backward compatibility. Use CLOB instead. Early versions – deprecated
LOB / binary LONG RAW Variable‑length binary data, max 2GB. For backward compatibility. Use BLOB instead. Early versions – deprecated
Rowid / special ROWID Base64 string representing the physical address of a row. Present in many versions
Rowid / special UROWID(size) Logical address of a row (for index‑organized tables). Max size 4000 bytes. Oracle8i (?) onwards
Rowid / special JSON (native) Stores JSON in optimized binary format (OSON). Oracle 21c
Rowid / special VECTOR Stores vector embeddings for AI Vector Search. Oracle 23ai

Version notes: Oracle5/6, Oracle7, Oracle8/8i, Oracle9i, Oracle10g, Oracle11g, Oracle12c, Oracle18c, Oracle19c, Oracle21c, Oracle23ai. “MAX_STRING_SIZE = EXTENDED” available from 12c onward. VARCHAR is currently identical to VARCHAR2 but use of VARCHAR2 is strongly recommended. Some very old types (LONG, LONG RAW) are kept only for backward compatibility — modern applications should use LOBs. Max sizes listed are typical; some depend on block size or national character set. Refer to official Oracle documentation for exact limits.
